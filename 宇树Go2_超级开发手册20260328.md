## 🌟 给初学者的话（零基础必读）

**欢迎来到机器人的世界！** 如果你是第一次接触编程或机器人开发，不用担心——这本手册会手把手教你。

### 我需要什么基础？

**完全零基础** 也可以开始！但建议你先了解以下内容（不懂也没关系，后面会教）：

| 你可能不懂的东西 | 简单解释 | 在哪里会用到 |
|-----------------|---------|------------|
| **Linux** | 一种电脑操作系统，像 Windows 一样，但机器人开发都用它 | 所有章节 |
| **终端/命令行** | 在电脑上输入文字命令来操控电脑（不用鼠标点） | 所有章节 |
| **Python** | 一种编程语言，像写英语作文一样写代码 | 第六章、第十四章等 |
| **C++** | 另一种编程语言，比 Python 快但更难写 | 第五章、第七章等 |
| **ROS2** | 机器人操作系统，帮多个程序协同工作来控制机器人 | 第十一章、第二十一章 |
| **LiDAR** | 激光雷达，用激光测量周围物体的距离，像蝙蝠的"超声波" | 第十四章、第二十章 |
| **SLAM** | 一边走一边画地图的技术，像你边走迷宫边画地图 | 第十四章、第二十章 |
| **SDK** | 软件开发工具包，别人写好的工具，让你更方便地控制机器人 | 第五章、第六章 |
| **DDS** | 数据分发服务，程序之间传递消息的"邮局" | 第三章 |

### 遇到问题怎么办？

1. **看报错信息** — 电脑给出的错误提示往往就是答案
2. **搜索引擎** — 把报错信息复制粘贴到百度/Google 搜索
3. **官方社区** — https://www.unifolm.com/
4. **GitHub Issues** — 在对应仓库提问
5. **重启试试** — 很多时候真的管用 😄

## 第一章 Go2 产品概述

### 1.1 产品定位

宇树 Go2 是宇树科技（Unitree Robotics）推出的**新一代交互仿生四足机器人**。作为 Go1 的全面升级版，Go2 在感知系统、运动能力、AI 智能和二次开发能力上都有显著提升。Go2 系列提供 AIR、PRO、X、EDU 四个版本，覆盖消费级到科研级需求。

### 1.3 核心亮点

- **4D 超广角激光雷达 L2**: 360° × 96° 半球形感知，最小探测距离 0.05m
- **AI 模式**: 通过大规模 AI 仿真训练，掌握倒立行走、自适应翻身起立、越障攀爬
- **ISS 2.0 智能伴随**: 无线矢量定位，定位精度提升 50%，遥控距离 >30m
- **3D 激光建图**: 使用自研 L2 雷达 + APP 实现点云地图构建与自主导航
- **暴走模式**: 充分释放关节电机潜能，复杂地形运动更快更灵活
- **OTA 升级**: 持续云端更新，不断进化

### 3.2 通信协议：DDS（CycloneDDS）

Go2 使用 **CycloneDDS**（版本 0.10.2）作为底层通信协议。DDS 同时也是 ROS2 的通信机制，因此 Go2 天然兼容 ROS2。

**关键概念**:

| 概念 | 说明 |
|------|------|
| **Topic** | 数据通道名称，如 `rt/sportmodestate` |
| **Publisher** | 数据发布者 |
| **Subscriber** | 数据订阅者 |
| **Domain ID** | DDS 域 ID，实机默认为 0，仿真建议为 1 |
| **IDL 类型** | Go2 使用 `unitree_go` IDL 类型 |

### 3.3 网络架构

| 设备 | IP 地址 | 说明 |
|------|---------|------|
| Go2 机载主板 | `192.168.123.161` | **固定不可修改** |
| 用户 PC | `192.168.123.222` | 用户自定义（推荐，"222"可改） |
| 扩展坞 Orin | 由 DHCP 或手动配置 | 通过背部 RJ45 连接 |

> ⚠️ 不允许将用户 PC 的 IP 设为 `192.168.123.161`，这是机器人内部地址。

## 第四章 开发环境搭建

> 🔰 **小白提示**：这一章教你如何准备好开发 Go2 所需要的电脑环境。就像画画前要准备好画笔和颜料一样，编程前也要先安装好各种工具。请按照步骤一步一步来。

### 4.1 系统要求

| 项目 | 要求 |
|------|------|
| **操作系统** | Ubuntu 20.04 或 Ubuntu 22.04 |
| **CPU 架构** | x86_64 或 aarch64 |
| **编译器** | GCC 9.4.0+ |
| **CMake** | 3.10+ |

> ⚠️ 不支持 Mac、Windows 系统开发。不允许在 Go2 内置电脑上开发。

### 4.2 安装基础依赖

```bash
sudo apt update
sudo apt install -y cmake gcc g++ build-essential \
    libeigen3-dev libyaml-cpp-dev libboost-all-dev \
    libspdlog-dev libfmt-dev
```

### 4.3 网络配置

#### 步骤 1: 物理连接

用网线连接 Go2 和你的电脑。

#### 步骤 2: 配置 IP

打开网络设置，找到连接机器人的网卡，将 IPv4 设为手动模式：

- **IP 地址**: `192.168.123.222`（"222"可自定义，但不能是 161）
- **子网掩码**: `255.255.255.0`

#### 步骤 3: 验证连接

```bash
ping 192.168.123.161
```

看到正常回应即表示连接成功。

#### 步骤 4: 查看网卡名

```bash
ifconfig
```

找到 IP 为 `192.168.123.222` 的网卡名（如 `enxf8e43b808e06`、`enp3s0` 等），**记住这个名字**，后续运行程序时需要。

### 4.4 安装 unitree_sdk2（C++）

```bash
cd ~
git clone https://github.com/unitreerobotics/unitree_sdk2.git
cd unitree_sdk2
mkdir build && cd build

# 安装到默认路径
cmake ..
sudo make install

# 或安装到指定路径
cmake .. -DCMAKE_INSTALL_PREFIX=/opt/unitree_robotics
sudo make install
```

### 4.5 安装 unitree_sdk2_python（Python）

```bash
cd ~
sudo apt install python3-pip
git clone https://github.com/unitreerobotics/unitree_sdk2_python.git
cd unitree_sdk2_python
```

## 第七章 低层电机控制

> 🔰 **小白提示**：低层控制就是直接操控 12 个关节电机，像操控提线木偶一样精确。这是**高级功能**，初学者建议先学第八章的高层控制（更安全、更简单）。

### 7.1 概述

低层控制直接操控 12 个关节电机，适用于：
- 自定义步态算法
- 强化学习策略部署
- 电机特性测试
- 研究型开发

> ⚠️ **运行低层控制前必须关闭 MCF 服务！** 否则两个控制源同时发送指令会导致机器人失控。

### 7.2 电机命令 (LowCmd)

通过发布 `unitree_go::msg::LowCmd` 消息到 `/lowcmd` Topic 来控制电机。

```cpp
// LowCmd 结构
struct LowCmd {
    uint8_t head[2];
    uint8_t level_flag;
    uint8_t frame_reserve;
    uint32_t sn[2];
    uint32_t version[2];
    uint16_t bandwidth;
    MotorCmd motor_cmd[20];  // 电机命令（使用前12个）
    BmsCmd bms_cmd;
    uint8_t wireless_remote[40];
    uint8_t led[12];
    uint8_t fan[2];
    uint8_t gpio;
    uint32_t reserve;
    uint32_t crc;
};
```

### 7.3 电机命令结构 (MotorCmd)

```cpp
struct MotorCmd {
    uint8_t mode;   // 模式: 0x01=FOC模式, 0x00=停止模式
    float q;        // 目标位置 (rad)
    float dq;       // 目标速度 (rad/s)
    float tau;      // 目标力矩 (N.m)
    float kp;       // 位置环增益
    float kd;       // 速度环增益
    unsigned long reserve[3];
};
```

**电机控制公式**:

$$\tau_{output} = kp \times (q_{target} - q_{current}) + kd \times (dq_{target} - dq_{current}) + \tau_{feedforward}$$

### 7.5 C++ 低层控制示例

```cpp
#include <unitree/idl/go2/LowCmd_.hpp>
#include <unitree/idl/go2/LowState_.hpp>
#include <unitree/robot/channel/channel_publisher.hpp>
#include <unitree/robot/channel/channel_subscriber.hpp>

#define TOPIC_LOWCMD "rt/lowcmd"
#define TOPIC_LOWSTATE "rt/lowstate"

using namespace unitree::robot;

// 状态回调
void LowStateHandler(const void* message) {
    auto state = *(unitree_go::msg::dds_::LowState_*)message;
    // 获取第0号电机（FR_hip）的状态
    float q = state.motor_state()[0].q();          // 角度
    float dq = state.motor_state()[0].dq();         // 角速度
    float tau = state.motor_state()[0].tau_est();    // 估计力矩
    int8_t temp = state.motor_state()[0].temperature(); // 温度
}

int main(int argc, char** argv) {
    ChannelFactory::Instance()->Init(0, argv[1]);
    
    // 创建发布者和订阅者
    ChannelPublisher<unitree_go::msg::dds_::LowCmd_> pub(TOPIC_LOWCMD);
    pub.InitChannel();
    
    ChannelSubscriber<unitree_go::msg::dds_::LowState_> sub(TOPIC_LOWSTATE);
    sub.InitChannel(LowStateHandler);
    
    // 构造控制命令
    unitree_go::msg::dds_::LowCmd_ cmd;
    cmd.motor_cmd()[0].mode() = 0x01;  // FOC 模式
    cmd.motor_cmd()[0].q() = 0.0;      // 目标角度
    cmd.motor_cmd()[0].dq() = 0.0;     // 目标角速度
    cmd.motor_cmd()[0].tau() = 1.0;    // 前馈力矩 1 N.m
    cmd.motor_cmd()[0].kp() = 0.0;
    cmd.motor_cmd()[0].kd() = 0.0;
    
    pub.Write(cmd);
    return 0;
}
```

### 7.6 安全提示

1. **首次测试时务必将机器人四腿悬空**
2. 先用小力矩测试，确认方向和范围正确
3. 设置合理的 kp/kd 增益，避免关节振荡
4. 随时准备好按急停或切断电源

### 8.11 姿态控制完整示例

```cpp
#include <cmath>
#include <signal.h>
#include <unistd.h>
#include <unitree/robot/go2/sport/sport_client.hpp>

bool stopped = false;

void sigint_handler(int sig) {
    if (sig == SIGINT) stopped = true;
}

int main(int argc, char** argv) {
    if (argc < 2) {
        std::cout << "Usage: " << argv[0] << " networkInterface" << std::endl;
        return -1;
    }
    
    unitree::robot::ChannelFactory::Instance()->Init(0, argv[1]);
    
    unitree::robot::go2::SportClient sport_client;
    sport_client.SetTimeout(10.0f);
    sport_client.Init();
    
    double t = 0, dt = 0.01;
    signal(SIGINT, sigint_handler);
    
    while (!stopped) {
        t += dt;
        // 姿态跟踪三角函数轨迹
        sport_client.Euler(0.2 * sin(2 * t), 0.2 * cos(2 * t) - 0.2, 0.0);
        sport_client.BalanceStand();
        usleep(int(dt * 1000000));
    }
    
    // 退出时复位姿态
    sport_client.Euler(0, 0, 0);
    return 0;
}
```

```cpp
#include <unitree/idl/go2/SportModeState_.hpp>
#include <unitree/robot/channel/channel_subscriber.hpp>

#define TOPIC_HIGHSTATE "rt/sportmodestate"

void HighStateHandler(const void* message) {
    auto state = *(unitree_go::msg::dds_::SportModeState_*)message;
    
    // 位置 (x, y, z)
    float x = state.position()[0];
    float y = state.position()[1];
    float z = state.position()[2];
    
    // 姿态四元数 (w, x, y, z)
    float qw = state.imu_state().quaternion()[0];
    float qx = state.imu_state().quaternion()[1];
    float qy = state.imu_state().quaternion()[2];
    float qz = state.imu_state().quaternion()[3];
    
    // 速度 (vx, vy, vz)
    float vx = state.velocity()[0];
    float vy = state.velocity()[1];
    float vz = state.velocity()[2];
    
    // 偏航角速度
    float yawSpeed = state.yaw_speed();
    
    // 体高
    float bodyHeight = state.body_height();
    
    // 足力
    int16_t footForce[4];
    for (int i = 0; i < 4; i++)
        footForce[i] = state.foot_force()[i];
    
    // 足端相对身体的位置 (12个值: 4腿 × 3坐标)
    for (int i = 0; i < 12; i++)
        float fp = state.foot_position_body()[i];
    
    // 足端相对身体的速度 (12个值)
    for (int i = 0; i < 12; i++)
        float fs = state.foot_speed_body()[i];
    
    // 障碍物距离 (4方向)
    for (int i = 0; i < 4; i++)
        float obs = state.range_obstacle()[i];
}

int main() {
    unitree::robot::ChannelFactory::Instance()->Init(0, "enp3s0");
    
    ChannelSubscriber<unitree_go::msg::dds_::SportModeState_> sub(TOPIC_HIGHSTATE);
    sub.InitChannel(HighStateHandler);

    return 0;
}
```

## 第十章 运动服务切换接口

### 10.1 概述

`MotionSwitcherClient` 用于在不同运动控制模式之间切换。这是运行低层控制程序前关闭默认运动服务的关键接口。

### 10.2 运动控制模式名称

| 名称 | 适用版本 |
|------|----------|
| `mcf` | 软件版本 ≥ 1.1.6 |
| `ai` (AI模式)、`normal` (普通)、`advanced` (前沿) | 软件版本 < 1.1.6 |

### 10.3 接口说明

```cpp
#include <unitree/robot/go2/motion_switcher/motion_switcher_client.hpp>

unitree::robot::go2::MotionSwitcherClient switcher;
switcher.SetTimeout(10.0f);
switcher.Init();

// 检查当前模式
std::string form, name;
switcher.CheckMode(form, name);
// form: "0"=标准形态, "1"=轮足形态
// name: 当前运动控制模式名

// 选择运动控制模式
switcher.SelectMode("mcf");  // 启用 MCF 服务

// 关闭运动控制模式（运行低层控制前调用）
switcher.ReleaseMode();

// 设置静默状态（Go2 启动时不自动启动默认运动控制）
switcher.SetSilent(true);   // 设置静默
switcher.SetSilent(false);  // 取消静默

// 获取静默状态
bool silent;
switcher.GetSilent(silent);
```

## 第十三章 强化学习训练 (unitree_rl_gym)

> 🔰 **小白提示**：强化学习就像训练小狗——做对了给奖励，做错了扣分。通过成千上万次试错，AI 自己学会了走路跑步甚至翻跟头。这一章教你如何用 AI 训练 Go2 学新技能。需要 NVIDIA 显卡。

### 13.1 概述

`unitree_rl_gym` 是宇树官方的强化学习训练框架，支持 Go2、G1、H1、H1-2 机器人。基于 Isaac Gym 和 legged_gym，提供完整的 **Train → Play → Sim2Sim → Sim2Real** 工作流。

**GitHub**: https://github.com/unitreerobotics/unitree_rl_gym (⭐3k)

### 13.2 工作流程

```
Train（Isaac Gym）→ Play（验证）→ Sim2Sim（MuJoCo）→ Sim2Real（实机部署）
```

1. **Train**: 在 Isaac Gym 仿真环境中训练，通过奖励函数学习运动策略
2. **Play**: 可视化验证策略是否符合预期
3. **Sim2Sim**: 将策略部署到 MuJoCo 仿真器，确保不过拟合
4. **Sim2Real**: 部署到物理机器人

### 13.3 安装

详细步骤参考: https://github.com/unitreerobotics/unitree_rl_gym/blob/main/doc/setup_en.md

主要依赖：
- **Isaac Gym**: https://developer.nvidia.com/isaac-gym
- **legged_gym**: 基础训练框架
- **rsl_rl**: 强化学习算法实现
- **MuJoCo**: Sim2Sim 验证
- **unitree_sdk2_python**: Sim2Real 部署

### 13.4 训练

```bash

    --max_iterations=10000 \  # 最大迭代次数
```

| `--sim_device` | 仿真设备，CPU: `--sim_device=cpu` |
| `--rl_device` | RL 设备，CPU: `--rl_device=cpu` |

#### 14.3.6 地图后处理

FAST-LIO2 输出的是 3D 点云地图（`.pcd`），需要转换为 2D 占用栅格地图才能供 Nav2 使用。

```python
#!/usr/bin/env python3
"""pcd_to_2d_map.py — 将 3D PCD 点云转换为 2D 占用栅格地图"""

import numpy as np
import open3d as o3d
import yaml
from PIL import Image

def pcd_to_gridmap(pcd_file, output_prefix, resolution=0.05,
                   z_min=0.1, z_max=1.5):
    """
    Args:
        pcd_file: PCD 文件路径
        output_prefix: 输出文件前缀 (会生成 .pgm 和 .yaml)
        resolution: 栅格分辨率 (m/pixel)，默认 5cm
        z_min: 最低高度截取 (m)，过滤地面
        z_max: 最高高度截取 (m)，过滤天花板/噪声
    """
    # 1. 读取点云
    pcd = o3d.io.read_point_cloud(pcd_file)
    points = np.asarray(pcd.points)
    print(f"原始点数: {len(points)}")

    # 2. 高度过滤 — 只保留特定高度范围的点（障碍物层）
    mask = (points[:, 2] >= z_min) & (points[:, 2] <= z_max)
    filtered = points[mask]
    print(f"过滤后点数: {len(filtered)}")

    # 3. 计算地图边界
    x_min, y_min = filtered[:, 0].min(), filtered[:, 1].min()
    x_max, y_max = filtered[:, 0].max(), filtered[:, 1].max()

    # 外扩 1m 边距
    margin = 1.0
    x_min -= margin; y_min -= margin
    x_max += margin; y_max += margin

    width = int(np.ceil((x_max - x_min) / resolution))
    height = int(np.ceil((y_max - y_min) / resolution))
    print(f"地图尺寸: {width} x {height} pixels")

    # 4. 生成栅格地图 (初始全部为未知=205)
    grid = np.full((height, width), 205, dtype=np.uint8)

    # 5. 将点云投影到 2D 栅格（障碍物=0/黑色）
    for p in filtered:
        col = int((p[0] - x_min) / resolution)
        row = int((p[1] - y_min) / resolution)
        row = height - 1 - row  # 翻转 y 轴
        if 0 <= row < height and 0 <= col < width:
            grid[row, col] = 0  # 占用

    # 6. 膨胀处理 — 对障碍物周围小范围标记为自由空间
    from scipy.ndimage import binary_dilation
    occupied = grid == 0
    # 将障碍物附近的未知区域标记为自由
    dilated = binary_dilation(occupied, iterations=3)
    free_mask = dilated & (~occupied) & (grid == 205)
    grid[free_mask] = 254  # 自由空间

    # 7. 保存地图图片 (.pgm)
    img = Image.fromarray(grid)
    pgm_file = f"{output_prefix}.pgm"
    img.save(pgm_file)

    # 8. 保存地图元数据 (.yaml)
    map_yaml = {
        'image': pgm_file,
        'resolution': resolution,
        'origin': [x_min, y_min, 0.0],
        'negate': 0,
        'occupied_thresh': 0.65,
        'free_thresh': 0.196
    }
    yaml_file = f"{output_prefix}.yaml"
    with open(yaml_file, 'w') as f:
        yaml.dump(map_yaml, f, default_flow_style=False)

    print(f"地图已保存: {pgm_file}, {yaml_file}")

if __name__ == "__main__":
    import sys
    if len(sys.argv) < 2:
        print("用法: python3 pcd_to_2d_map.py <input.pcd> [output_prefix]")
        sys.exit(1)
    pcd_file = sys.argv[1]
    output = sys.argv[2] if len(sys.argv) > 2 else "map"
    pcd_to_gridmap(pcd_file, output)
```

#### 14.4.5 cmd_vel 桥接节点

Nav2 输出 `/cmd_vel`（geometry_msgs/Twist），需要桥接到 Go2 的 `SportClient`：

```python
#!/usr/bin/env python3
"""cmd_vel_bridge.py — 将 Nav2 的 /cmd_vel 转换为 Go2 运动指令"""

import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist

from unitree_sdk2py.core.channel import ChannelFactoryInitialize
from unitree_sdk2py.go2.sport.sport_client import SportClient


class CmdVelBridge(Node):
    def __init__(self):
        super().__init__('cmd_vel_bridge')

        # 初始化 Go2 SDK
        ChannelFactoryInitialize(0, "eth0")  # 替换为实际网卡名
        self.sport_client = SportClient()
        self.sport_client.SetTimeout(5.0)
        self.sport_client.Init()

        # 订阅 Nav2 的 /cmd_vel
        self.sub = self.create_subscription(
            Twist, '/cmd_vel', self.cmd_vel_callback, 10)

        # 安全超时：如果 0.5s 没收到新指令，停止运动
        self.timer = self.create_timer(0.5, self.safety_timeout)
        self.last_cmd_time = self.get_clock().now()

        self.get_logger().info('Go2 cmd_vel bridge started')

    def cmd_vel_callback(self, msg: Twist):
        vx = msg.linear.x       # 前进速度 m/s
        vy = msg.linear.y       # 横移速度 m/s
        vyaw = msg.angular.z    # 偏航角速度 rad/s

        # 限幅保护
        vx = max(-1.0, min(vx, 2.0))
        vy = max(-0.5, min(vy, 0.5))
        vyaw = max(-2.0, min(vyaw, 2.0))

        self.sport_client.Move(vx, vy, vyaw)
        self.last_cmd_time = self.get_clock().now()

    def safety_timeout(self):
        elapsed = (self.get_clock().now() - self.last_cmd_time).nanoseconds / 1e9
        if elapsed > 0.5:
            self.sport_client.StopMove()


def main():
    rclpy.init()
    node = CmdVelBridge()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

#### 14.4.6 TF 树配置

Nav2 需要完整的 TF 树。创建静态 TF 发布：

```python
#!/usr/bin/env python3
"""go2_tf_publisher.py — 发布 Go2 的 TF 树"""

import rclpy
from rclpy.node import Node
from tf2_ros import StaticTransformBroadcaster, TransformBroadcaster
from geometry_msgs.msg import TransformStamped
import math

from unitree_sdk2py.core.channel import ChannelFactoryInitialize
# 订阅 Go2 里程计


class Go2TFPublisher(Node):
    def __init__(self):
        super().__init__('go2_tf_publisher')
        
        # 静态 TF: base_link → lidar_link
        self.static_br = StaticTransformBroadcaster(self)
        t = TransformStamped()
        t.header.stamp = self.get_clock().now().to_msg()
        t.header.frame_id = 'base_link'
        t.child_frame_id = 'lidar_link'
        # MID-360 安装在 Go2 背部，根据实际安装位置调整
        t.transform.translation.x = 0.0
        t.transform.translation.y = 0.0
        t.transform.translation.z = 0.45  # 高度约 45cm
        t.transform.rotation.w = 1.0
        self.static_br.sendTransform(t)
        
        # 动态 TF: odom → base_link (由里程计更新)
        self.br = TransformBroadcaster(self)
        self.timer = self.create_timer(0.02, self.publish_odom_tf)  # 50Hz
```

```python
#!/usr/bin/env python3
"""go2_nav_to_pose.py — 编程方式发送导航目标"""

import rclpy
from rclpy.node import Node
from rclpy.action import ActionClient
from nav2_msgs.action import NavigateToPose
from geometry_msgs.msg import PoseStamped
import math


class Go2Navigator(Node):
    def __init__(self):
        super().__init__('go2_navigator')
        self.nav_client = ActionClient(
            self, NavigateToPose, 'navigate_to_pose')

    def go_to(self, x, y, yaw=0.0):
        """导航到指定坐标点"""
        self.nav_client.wait_for_server()

        goal = NavigateToPose.Goal()
        goal.pose = PoseStamped()
        goal.pose.header.frame_id = 'map'
        goal.pose.header.stamp = self.get_clock().now().to_msg()
        goal.pose.pose.position.x = x
        goal.pose.pose.position.y = y
        goal.pose.pose.position.z = 0.0
        # yaw → quaternion
        goal.pose.pose.orientation.z = math.sin(yaw / 2)
        goal.pose.pose.orientation.w = math.cos(yaw / 2)

        self.get_logger().info(f'导航目标: ({x}, {y}, yaw={yaw:.2f})')
        future = self.nav_client.send_goal_async(
            goal, feedback_callback=self.feedback_cb)
        future.add_done_callback(self.goal_response_cb)

    def goal_response_cb(self, future):
        goal_handle = future.result()
        if not goal_handle.accepted:
            self.get_logger().warn('目标被拒绝')
            return
        self.get_logger().info('目标已接受，正在导航...')
        result_future = goal_handle.get_result_async()
        result_future.add_done_callback(self.result_cb)

    def feedback_cb(self, feedback_msg):
        pos = feedback_msg.feedback.current_pose.pose.position
        self.get_logger().info(f'当前位置: ({pos.x:.2f}, {pos.y:.2f})')

    def result_cb(self, future):
        self.get_logger().info('导航完成!')


def main():
    rclpy.init()
    nav = Go2Navigator()
    nav.go_to(3.0, 2.0, yaw=1.57)  # 导航到 (3, 2)，面朝北
    rclpy.spin(nav)
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

```python
# ───── 数据模型 ─────

class NavGoalRequest(BaseModel):
    x: float
    y: float
    yaw: float = 0.0

class MoveRequest(BaseModel):
    vx: float = 0.0     # 前进速度 m/s
    vy: float = 0.0     # 横移速度 m/s
    vyaw: float = 0.0   # 偏航角速度 rad/s

class ActionRequest(BaseModel):
    action: str          # hello, sit, stand, dance1, dance2, stretch 等

# ───── ROS2 节点 ─────

class Go2APINode(Node):
    def __init__(self):
        super().__init__('go2_api_node')

        # Nav2 Action Client
        self.nav_client = ActionClient(
            self, NavigateToPose, 'navigate_to_pose')

        # cmd_vel 发布者（手动控制时使用）
        self.cmd_pub = self.create_publisher(Twist, '/cmd_vel', 10)

        # 状态订阅
        self.robot_state = {
            'position': {'x': 0.0, 'y': 0.0, 'z': 0.0},
            'velocity': {'vx': 0.0, 'vy': 0.0, 'vz': 0.0},
            'battery': 100.0,
            'mode': 'idle',
        }
        self.nav_status = {
            'active': False,
            'goal': None,
            'progress': 0.0,
            'feedback_pose': None,
        }
        self._goal_handle = None

        # 订阅 Go2 相关 Topic (根据实际环境适配)
        # self.state_sub = self.create_subscription(...)

    # ── 导航 ──
    async def navigate_to(self, x: float, y: float, yaw: float):
        if not self.nav_client.wait_for_server(timeout_sec=5.0):
            return {'success': False, 'message': 'Nav2 服务未就绪'}

        goal = NavigateToPose.Goal()
        goal.pose = PoseStamped()
        goal.pose.header.frame_id = 'map'
        goal.pose.header.stamp = self.get_clock().now().to_msg()
        goal.pose.pose.position.x = x
        goal.pose.pose.position.y = y
        goal.pose.pose.orientation.z = math.sin(yaw / 2)
        goal.pose.pose.orientation.w = math.cos(yaw / 2)

        future = self.nav_client.send_goal_async(
            goal, feedback_callback=self._nav_feedback_cb)
        goal_handle = await asyncio.wrap_future(
            asyncio.ensure_future(self._await_goal(future)))

        if not goal_handle or not goal_handle.accepted:
            self.nav_status['active'] = False
            return {'success': False, 'message': '导航目标被拒绝'}

        self._goal_handle = goal_handle
        self.nav_status.update({
            'active': True,
            'goal': {'x': x, 'y': y, 'yaw': yaw},
            'progress': 0.0,
        })
        return {'success': True, 'message': f'导航至 ({x:.2f}, {y:.2f})'}

    async def _await_goal(self, future):
        """在 asyncio 中等待 ROS2 future"""
        loop = asyncio.get_event_loop()
        result = await loop.run_in_executor(None, lambda: future.result())
        return result

    def _nav_feedback_cb(self, feedback_msg):
        pos = feedback_msg.feedback.current_pose.pose.position
        self.nav_status['feedback_pose'] = {
            'x': pos.x, 'y': pos.y
        }

    async def cancel_navigation(self):
        if self._goal_handle:
            self._goal_handle.cancel_goal_async()
            self._goal_handle = None
        self.nav_status['active'] = False
        return {'success': True, 'message': '导航已取消'}

    # ── 手动控制 ──
    def send_move(self, vx, vy, vyaw):
        # 限幅保护
        vx = max(-1.0, min(vx, 2.0))
        vy = max(-0.5, min(vy, 0.5))
        vyaw = max(-2.0, min(vyaw, 2.0))

        msg = Twist()
        msg.linear.x = vx
        msg.linear.y = vy
        msg.angular.z = vyaw
        self.cmd_pub.publish(msg)

    def stop_move(self):
        msg = Twist()  # 全零速度
        self.cmd_pub.publish(msg)


# ───── 全局实例 ─────

    ros_thread.start()

    ros_node.destroy_node()
    rclpy.shutdown()
```

```javascript
    // 1. 获取地图元信息

        const northEast = L.latLng(origin[1] + h, origin[0] + w);

        map.fitBounds(bounds);

        L.marker([y, x]).addTo(map)
            .bindPopup(`目标: (${x.toFixed(2)}, ${y.toFixed(2)})`)
            .openPopup();

        // 发送导航请求
        await fetch(`${API}/nav/goal`, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ x, y, yaw: 0 }),
        });
    });

    // 6. 实时更新机器人位置
    const ws = new WebSocket('ws://192.168.123.100:8080/ws/status');
    ws.onmessage = (e) => {
        const data = JSON.parse(e.data);
        const pos = data.robot.position;
        robotMarker.setLatLng([pos.y, pos.x]);
    };

    return map;
}
```

### 14.6 多点巡航

#### 14.6.1 巡航 API

在 `go2_api_server.py` 中添加多点巡航端点：

```python
class PatrolRequest(BaseModel):
    waypoints: list[NavGoalRequest]   # 巡航路径点列表
    loop: bool = False                # 是否循环巡航

patrol_task = None  # 巡航协程句柄

@app.post("/nav/patrol", tags=["导航"])
async def start_patrol(req: PatrolRequest):
    """多点巡航 — 按顺序导航到各个路径点"""
    global patrol_task

    async def _run_patrol():
        while True:
            for i, wp in enumerate(req.waypoints):
                ros_node.get_logger().info(
                    f'巡航点 {i+1}/{len(req.waypoints)}: '
                    f'({wp.x}, {wp.y})')
                await ros_node.navigate_to(wp.x, wp.y, wp.yaw)

                # 等待导航完成
                while ros_node.nav_status['active']:
                    await asyncio.sleep(0.5)

            if not req.loop:
                break

    patrol_task = asyncio.create_task(_run_patrol())
    return {
        "success": True,
        "message": f"开始巡航，共 {len(req.waypoints)} 个路径点，"
                   f"循环={'是' if req.loop else '否'}"
    }

@app.post("/nav/patrol/stop", tags=["导航"])
async def stop_patrol():
    """停止巡航"""
    global patrol_task
    if patrol_task:
        patrol_task.cancel()
        patrol_task = None
    await ros_node.cancel_navigation()
    return {"success": True, "message": "巡航已停止"}
```

### 14.8 导航开发最佳实践

1. **建图**：移动速度 ≤0.5m/s，尽量回环闭合，避免纯旋转
2. **定位**：AMCL 适合 2D 场景，FAST-LIO2 适合 3D 复杂场景
3. **Nav2 调参**：先调 costmap 膨胀半径（`robot_radius` + 余量），再调 controller 速度限制
4. **安全**：`cmd_vel_bridge` 中务必加限幅保护和超时停止
5. **前端**：使用 WebSocket 实时推送状态，避免高频 HTTP 轮询
6. **部署**：API 服务运行在 Orin NX 上，前端可部署在任意 Web 服务器或直接通过 Nginx 代理
7. **多机**：若需控制多台 Go2，可通过不同的 DDS Domain ID 或 ROS2 Namespace 隔离

## 第十八章 安全须知与最佳实践

### 18.1 首次使用注意事项

1. **充分阅读官方文档**，了解各版本功能差异

3. 从小力矩开始测试，逐步加大

5. 不要同时运行多个控制程序

### 18.3 高层控制安全

1. 特殊动作（翻转、跳跃）需要在**平坦宽敞区域**执行

### 18.5 电池安全

2. 不在高温/低温环境下充电
3. 低电量警告（黄色慢闪）后及时充电

### 18.6 机械安全

## 第二十章 4D LiDAR L2 开发与 Point-LIO SLAM

### 20.1 什么是 4D LiDAR？（小白必读）

> 🔰 **完全不懂也没关系，看完这一节你就明白了！**

**LiDAR**（Light Detection And Ranging，光探测与测距）就像蝙蝠的"超声波"——蝙蝠发出超声波，通过回声判断周围物体的距离和位置。LiDAR 用的不是声波，而是**激光**。

```
普通 3D LiDAR:  发射激光 → 测量距离 → 得到 (x, y, z) 三维坐标
4D LiDAR:       发射激光 → 测量距离 + 反射强度 → 得到 (x, y, z, 灰度) 四维数据
                                                    ↑ 多了一个"灰度"维度
```

**为什么叫"4D"？**

| 维度 | 含义 | 类比 |
|------|------|------|
| X | 前后距离 | 你往前走了几步 |
| Y | 左右距离 | 你往左边站了几步 |
| Z | 上下距离 | 你站在几楼 |
| 灰度 | 物体表面反射强度 | 这个东西是亮的还是暗的 |

有了灰度信息，机器人不仅能知道"那里有个东西"，还能知道"那个东西是什么材质"——比如玻璃的反射率低，白墙的反射率高。

**Go2 上的 LiDAR**：
- Go2 头部自带一个 **L1** LiDAR，主要用于避障
- 你可以额外购买并安装 **L2** LiDAR，用于建图、导航等高级功能
- L2 比 L1 更强大，支持以太网通信，精度更高

| **最小探测距离** | 0.05m | 0.05m | 相同 |
| **尺寸/重量** | 75×75×65mm / 230g | 75×75×65mm / 230g | 完全相同 |

### 20.3 L2 接口与连接

#### 20.3.1 L2 接口说明

L2 提供两种通信方式:

```
┌────────────────────────────────────────────┐
│           Unitree LiDAR L2                  │
│                                            │
│   ┌──────────┐        ┌──────────────┐    │
│   │ 以太网口  │        │  串口(UART)   │    │
│   │ (推荐✓)  │        │  (兼容L1)     │    │
│   └─────┬────┘        └──────┬───────┘    │
│         │                    │             │
│    100Mbps               115200bps        │
│    大数据量               小数据量          │
│    实时性好               延迟稍高          │
└─────────┼────────────────────┼─────────────┘
          │                    │
          ▼                    ▼
   ┌──────────┐        ┌──────────┐
   │   电脑    │        │   电脑    │
   │ (网口RJ45)│       │ (USB转TTL)│
   └──────────┘        └──────────┘
```

#### 20.3.2 以太网连接（推荐）

这是 L2 相比 L1 最重要的升级。以太网连接速度快、稳定、数据量大。

**第一步：物理连接**
```
L2 LiDAR 网口 ──── 网线(RJ45) ────→ 电脑/Orin NX 网口
L2 LiDAR 供电 ──── 12V DC 电源
```

**第二步：配置电脑 IP**
```bash
# 查看网卡名称（找到连接 L2 的那个网口）
ifconfig
# 通常是 eth0 或 enp3s0 等

# 设置 IP（L2 默认 IP 是 192.168.1.100）
sudo ifconfig eth0 192.168.1.50 netmask 255.255.255.0

# 测试连接
ping 192.168.1.100
# 如果 ping 通了就说明连接成功！
```

**第三步：验证连接**
```bash
# L2 默认 IP: 192.168.1.100
# 如果 ping 通，你会看到类似这样的输出：
# 64 bytes from 192.168.1.100: icmp_seq=1 ttl=64 time=0.5 ms
```

#### 20.3.3 串口连接（备选）

如果你的电脑没有网口，可以用串口连接：

```bash
# 需要一个 USB 转 TTL 模块
# 连接：L2 的 TX → USB转TTL 的 RX
#       L2 的 RX → USB转TTL 的 TX
#       L2 的 GND → USB转TTL 的 GND

# 查看串口设备

# 通常是 /dev/ttyUSB0

# 设置串口权限
```

### 20.5 L2 点云数据获取（编程）

#### 20.5.1 Python 读取 L2 点云

```python
#!/usr/bin/env python3
"""
read_l2_pointcloud.py — 用 Python 读取 L2 LiDAR 的点云数据

这个程序会：
1. 连接到 L2 LiDAR
2. 接收点云数据
3. 打印出每个点的坐标

运行前请先启动 L2 驱动：
  roslaunch unitree_lidar_ros run_without_rviz.launch
"""

import rospy                          # ROS 的 Python 库
from sensor_msgs.msg import PointCloud2  # 点云消息类型
import sensor_msgs.point_cloud2 as pc2   # 点云工具

def pointcloud_callback(msg):
    """
    每当收到新的点云数据时，这个函数就会被调用
    msg 就是 L2 发来的点云数据
    """
    # 把点云消息转换成一个个点的列表
    points = list(pc2.read_points(msg, field_names=("x", "y", "z", "intensity")))

    # 打印前 5 个点的信息
    print(f"\n--- 收到 {len(points)} 个点 ---")
    for i, point in enumerate(points[:5]):
        x, y, z, intensity = point
        print(f"  点 {i}: x={x:.3f}m, y={y:.3f}m, z={z:.3f}m, 灰度={intensity:.1f}")

def main():
    # 初始化 ROS 节点（给这个程序起个名字）
    rospy.init_node('l2_pointcloud_reader')

    # 订阅 L2 的点云 Topic
    # 当有新数据到来时，自动调用 pointcloud_callback 函数
    rospy.Subscriber('/unilidar/cloud', PointCloud2, pointcloud_callback)

    print("等待 L2 LiDAR 点云数据...")
    print("（如果长时间没有数据，请检查 L2 驱动是否已启动）")

    # 保持程序运行，持续接收数据
    rospy.spin()

if __name__ == '__main__':
    main()
```

#### 20.5.2 点云数据格式说明

```
每个点的数据结构：
┌──────────────────────────────────────────────────────┐
│  x (float32)  │  y (float32)  │  z (float32)  │ intensity (float32) │
│  前后距离(m)   │  左右距离(m)   │  上下距离(m)   │  反射强度(0~255)    │
└──────────────────────────────────────────────────────┘

L2 每秒产生约 64,000 个这样的点
一"帧"点云 ≈ 64000 / 5.55 ≈ 11,500 个点（5.55Hz 一圈）
```

### 20.6 Point-LIO SLAM 建图

> 🔰 **什么是 SLAM？** SLAM（Simultaneous Localization and Mapping）= 同时定位与建图。想象你走进一个完全黑暗的房间，你一边走一边用手电筒照，同时在脑子里画出这个房间的地图。SLAM 就是让机器人自动做这件事！

**Point-LIO** 是宇树官方推荐的 SLAM 算法，专门为 L1/L2 LiDAR 优化。它的特点是：
- **高带宽**：即使 LiDAR 快速移动也能准确建图

- **官方支持**：宇树官方维护，与 L1/L2 完美配合

#### 20.8.3 在 ROS2 中使用 L2 数据

```python
#!/usr/bin/env python3
"""
ros2_l2_subscriber.py — 在 ROS2 中接收 L2 点云数据

使用方法:
  1. 先启动 ROS1 L2 驱动和 ros1_bridge
  2. 然后运行: ros2 run your_package ros2_l2_subscriber
"""

import rclpy
from rclpy.node import Node
from sensor_msgs.msg import PointCloud2
from sensor_msgs_py import point_cloud2 as pc2  # ROS2 版点云工具


class L2Subscriber(Node):
    def __init__(self):
        super().__init__('l2_subscriber')  # 节点名称

        # 订阅 L2 点云 Topic（通过 Bridge 从 ROS1 转过来的）
        self.subscription = self.create_subscription(
            PointCloud2,          # 消息类型
            '/unilidar/cloud',    # Topic 名称
            self.listener_callback,  # 收到消息时调用的函数
            10                    # 消息缓冲区大小
        )
        self.get_logger().info('等待 L2 点云数据...')

    def listener_callback(self, msg):
        """收到 L2 点云数据时的处理函数"""
        # 获取点云中的点数
        point_count = msg.width * msg.height
        self.get_logger().info(f'收到 {point_count} 个点')

        # 你可以在这里处理点云数据，比如：
        # - 检测障碍物
        # - 计算最近物体距离
        # - 发送给导航系统


def main(args=None):
    rclpy.init(args=args)
    node = L2Subscriber()
    rclpy.spin(node)      # 保持运行
    node.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

| 建议 | 说明 |
|------|------|
| **优先用以太网** | 以太网比串口快 100 倍以上，点云数据量大时必须用以太网 |
| **IMU 初始化** | 启动后前 3-5 秒保持 L2 静止不动 |
| **避免遮挡** | 安装时确保 360° 视场不被遮挡太多 |
| **缓慢移动** | 建图速度 ≤ 0.5m/s，快了容易建图质量差 |
| **回环闭合** | 建图最后回到起点附近，大幅提升地图精度 |
| **温度注意** | L2 支持 -10°C~40°C，低温可开启自发热模式 |
| **数据保存** | 每次建图的 PCD 文件记得备份 |
| **先仿真再实机** | 没有硬件时先用官方 rosbag 数据集练习 |

### 20.10 L2 常见问题

**Q: L2 连上了但收不到点云数据？**

A: 检查以下几点：
1. IP 配置是否正确（L2 默认 192.168.1.100）
2. 电脑 IP 是否在同一网段（如 192.168.1.50）
3. 防火墙是否关闭：`sudo ufw disable`
4. 网线是否插紧

**Q: Point-LIO 建图效果很差，地图歪歪扭扭？**

A: 常见原因：
1. 移动太快——放慢速度到 0.3m/s
2. 环境特征太少（如空旷的走廊）——换更复杂的环境试试
3. IMU 没初始化——启动后静止 5 秒再移动
4. L2 安装不牢——检查是否有晃动

**Q: L1 和 L2 的 SDK 冲突了怎么办？**

A: L1 用 `unilidar_sdk`，L2 用 `unilidar_sdk2`，两个 SDK 独立，不会冲突。但运行时只能同时用一个 LiDAR 驱动节点。

## 第二十一章 ROS2 从零入门完全教程

> 🔰 **本章目标**：就算你从来没听说过 ROS，看完这一章也能理解并使用 ROS2 来开发 Go2 机器人。每个概念都会从"这是什么"开始讲起，配合大量代码示例和中文注释。

### 21.1 什么是 ROS2？（小白必读）

#### 21.1.1 用最简单的话解释 ROS2

**ROS**（Robot Operating System）= 机器人操作系统。但它**不是**像 Windows 那样的操作系统，而是一套**工具和框架**，帮你更方便地开发机器人程序。

**类比**：想象你在搭乐高积木——

| 不用 ROS | 用 ROS |
|----------|--------|
| 每个零件都要自己从零开始做 | 很多零件别人已经做好了，你直接拿来用 |
| 各个零件之间不兼容 | 所有零件用统一标准，随便组合 |
| 换一个机器人要全部重写 | 大部分代码可以复用 |

**ROS2** 是 ROS 的升级版（ROS1 的下一代），改进了：

| 对比 | ROS1 | ROS2 |
|------|------|------|
| **通信方式** | 自研（需要 roscore 中心节点） | 标准 DDS（去中心化，更可靠） |
| **实时性** | 差 | 好（支持实时系统） |
| **安全性** | 无 | 支持加密、认证 |
| **跨平台** | 主要 Linux | Linux + Windows + macOS |
| **Python 支持** | Python 2 | Python 3 |
| **Go2 使用** | ✗ | ✓ Go2 底层就是用 DDS 通信 |

> 💡 **关键点**：Go2 机器人底层使用 **CycloneDDS** 通信，而 ROS2 也使用 DDS，所以它们**天然兼容**！你可以直接用 ROS2 命令来和 Go2 对话。

#### 21.1.2 ROS2 能帮你做什么？

```
不用 ROS2 开发 Go2：
  你需要自己写 DDS 通信代码 → 自己解析消息 → 自己管理多个程序 → 很累

用 ROS2 开发 Go2：
  ros2 topic echo /sportmodestate    ← 一行命令就能看到机器人状态
  ros2 topic pub /cmd_vel ...         ← 一行命令就能控制机器人移动
  ros2 launch xxx                     ← 一键启动所有需要的程序
  rviz2                               ← 图形化界面看点云、地图、轨迹
```

### 21.2 ROS2 核心概念图解

在学习具体操作之前，先了解 ROS2 的几个核心概念。别急着记，后面用到的时候自然就懂了。

```
ROS2 核心概念关系图:

  ┌─────────────────────────────────────────────────────────┐
  │                   ROS2 系统                              │
  │                                                         │
  │   ┌──────────┐    Topic(话题)     ┌──────────┐        │
  │   │  节点 A   │───── 消息 ──────►│  节点 B   │        │
  │   │ (Node A)  │  (单向广播)       │ (Node B)  │        │
  │   │          │                   │          │        │
  │   │ 发布者    │                   │ 订阅者    │        │
  │   │Publisher │                   │Subscriber│        │
  │   └──────────┘                   └──────────┘        │
  │                                                       │
  │   ┌──────────┐   Service(服务)   ┌──────────┐        │
  │   │  节点 C   │◄──── 请求 ──────│  节点 D   │        │
  │   │ (Server)  │───── 响应 ──────►│ (Client)  │        │
  │   │ 服务端    │  (一问一答)       │ 客户端    │        │
  │   └──────────┘                   └──────────┘        │
  │                                                       │
  │   ┌──────────┐    Action(动作)   ┌──────────┐        │
  │   │  节点 E   │◄──── 目标 ──────│  节点 F   │        │
  │   │ (Server)  │───── 反馈 ──────►│ (Client)  │        │
  │   │ 动作服务端│───── 结果 ──────►│ 动作客户端│        │
  │   │          │ (长时间任务+进度)  │          │        │
  │   └──────────┘                   └──────────┘        │
  └─────────────────────────────────────────────────────────┘
```

#### 概念对照表

| 概念 | 类比 | 说明 | Go2 中的例子 |
|------|------|------|-------------|
| **节点 (Node)** | 一个工人 | 一个独立运行的程序 | LiDAR 驱动节点、导航节点 |
| **话题 (Topic)** | 广播频道 | 节点间传递消息的通道 | `/sportmodestate`（机器人状态） |
| **消息 (Message)** | 广播内容 | 在 Topic 上传递的数据 | 位置、速度、点云等 |
| **发布者 (Publisher)** | 电台主播 | 往 Topic 上发消息的 | Go2 发布自己的状态 |
| **订阅者 (Subscriber)** | 收音机听众 | 从 Topic 上收消息的 | 你的程序接收 Go2 状态 |
| **服务 (Service)** | 打电话 | 一问一答，有去有回 | 查询电池电量 |
| **动作 (Action)** | 叫外卖 | 发起请求→持续反馈进度→最终结果 | 导航到某个点 |
| **参数 (Parameter)** | 设置/配置 | 节点的可调参数 | 最大速度、安全距离 |
| **Launch 文件** | 剧本 | 一键启动多个节点 | 导航 launch 同时启动 5 个节点 |
| **包 (Package)** | 工具箱 | 一组相关代码的集合 | `unitree_go` 消息包 |
| **工作空间 (Workspace)** | 工作台 | 存放多个包的目录 | `~/ros2_ws/` |

### 21.7 编写你的第一个发布者（Publisher）

发布者就是一个往 Topic 上"广播"消息的程序。下面写一个最简单的发布者——每隔 0.5 秒发送一条 "Hello Go2!" 消息。

#### 21.7.1 Python 版本

在 `my_first_package/my_first_package/` 目录下创建 `my_publisher.py`：

```python
#!/usr/bin/env python3
"""
my_publisher.py — 我的第一个 ROS2 发布者

功能：每 0.5 秒往 /chatter Topic 上发送一条消息

学习要点：
  1. 如何创建一个 ROS2 节点
  2. 如何创建一个发布者
  3. 如何使用定时器周期性发布消息
"""

# 导入 ROS2 Python 库
import rclpy                      # ROS2 的 Python 客户端库
from rclpy.node import Node       # 所有 ROS2 节点的基类
from std_msgs.msg import String   # 标准字符串消息类型


class MyPublisher(Node):
    """我的第一个发布者节点"""

    def __init__(self):
        # 调用父类构造函数，给这个节点起名叫 'my_publisher'
        super().__init__('my_publisher')

        # 创建一个发布者
        # 参数1: String = 消息类型（字符串）
        # 参数2: 'chatter' = Topic 名称
        # 参数3: 10 = 消息缓冲区大小
        self.publisher_ = self.create_publisher(String, 'chatter', 10)

        # 创建一个定时器，每 0.5 秒触发一次
        timer_period = 0.5  # 秒
        self.timer = self.create_timer(timer_period, self.timer_callback)

        # 消息计数器
        self.count = 0

        # 打印启动信息
        self.get_logger().info('发布者节点启动！每 0.5 秒发送一条消息')

    def timer_callback(self):
        """定时器回调函数 — 每 0.5 秒自动执行一次"""
        # 创建消息
        msg = String()
        msg.data = f'Hello Go2! 第 {self.count} 条消息'

        # 发布消息
        self.publisher_.publish(msg)

        # 在终端打印日志
        self.get_logger().info(f'发布: "{msg.data}"')

        self.count += 1


def main(args=None):
    # 初始化 ROS2
    rclpy.init(args=args)

    # 创建节点实例
    node = MyPublisher()

    # 让节点保持运行（持续处理回调）
    # spin() 会一直运行，直到你按 Ctrl+C
    rclpy.spin(node)

    # 清理（Ctrl+C 后执行）
    node.destroy_node()
    rclpy.shutdown()


# Python 程序入口点
if __name__ == '__main__':
    main()
```

注册节点入口。编辑 `setup.py`，在 `entry_points` 中添加：

```python
entry_points={
    'console_scripts': [
        'my_publisher = my_first_package.my_publisher:main',
        # 格式：'可执行名 = 包名.文件名:函数名'
    ],
},
```

#### 21.7.2 编译并运行

```bash

colcon build --packages-select my_first_package
# --packages-select 只编译指定的包（更快）

# 加载

# 运行！

# 你应该看到:

# [INFO] [my_publisher]: 发布: "Hello Go2! 第 0 条消息"
# [INFO] [my_publisher]: 发布: "Hello Go2! 第 1 条消息"
# ...
```

### 21.8 编写你的第一个订阅者（Subscriber）

订阅者就是一个从 Topic 上"收听"消息的程序。

#### 21.8.1 Python 版本

创建 `my_first_package/my_first_package/my_subscriber.py`：

```python
#!/usr/bin/env python3
"""
my_subscriber.py — 我的第一个 ROS2 订阅者

功能：监听 /chatter Topic，收到消息就打印出来

学习要点：
  1. 如何创建订阅者
  2. 如何编写回调函数处理收到的消息
"""

import rclpy
from rclpy.node import Node
from std_msgs.msg import String


class MySubscriber(Node):
    """我的第一个订阅者节点"""

    def __init__(self):
        super().__init__('my_subscriber')

        # 创建订阅者
        # 参数1: String = 消息类型
        # 参数2: 'chatter' = Topic 名称（要和发布者一致！）
        # 参数3: self.listener_callback = 收到消息时调用的函数
        # 参数4: 10 = 消息缓冲区大小
        self.subscription = self.create_subscription(
            String, 'chatter', self.listener_callback, 10)

        self.get_logger().info('订阅者节点启动！等待消息...')

    def listener_callback(self, msg):
        """
        收到消息时的回调函数
        msg: 收到的消息对象
        """
        self.get_logger().info(f'收到: "{msg.data}"')


def main(args=None):
    rclpy.init(args=args)
    node = MySubscriber()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

### 21.10 服务（Service）

> 🔰 **Topic vs Service**：
> - **Topic** 像广播电台——一直在播，谁想听谁听
> - **Service** 像打电话——你问一个问题，对方回答一个答案

#### 21.10.1 定义服务类型

创建 `my_interfaces/srv/GetBattery.srv`：

```
# GetBattery.srv — 查询电池电量服务
# --- 上面是请求（Request），下面是响应（Response）
---
float32 battery_percentage    # 电池百分比
float32 voltage               # 电压 (V)
bool is_charging              # 是否在充电
```

在 `CMakeLists.txt` 中添加：

```cmake
rosidl_generate_interfaces(${PROJECT_NAME}
  "msg/RobotStatus.msg"
  "srv/GetBattery.srv"
  DEPENDENCIES std_msgs geometry_msgs
)
```

#### 21.10.2 编写服务端

```python
#!/usr/bin/env python3
"""battery_server.py — 电池查询服务端"""

import rclpy
from rclpy.node import Node
from my_interfaces.srv import GetBattery


class BatteryServer(Node):
    def __init__(self):
        super().__init__('battery_server')

        # 创建服务
        self.srv = self.create_service(
            GetBattery,          # 服务类型
            'get_battery',       # 服务名称
            self.handle_request  # 处理请求的函数
        )
        self.get_logger().info('电池查询服务已启动')

    def handle_request(self, request, response):
        """处理客户端的请求"""
        # 这里模拟返回电池数据
        # 实际使用中，你会从 Go2 传感器读取真实数据
        response.battery_percentage = 85.0
        response.voltage = 25.2
        response.is_charging = False
        self.get_logger().info(f'收到查询请求，返回电量: {response.battery_percentage}%')
        return response


def main():
    rclpy.init()
    node = BatteryServer()
    rclpy.spin(node)
    rclpy.shutdown()
```

#### 21.10.3 编写客户端

```python
#!/usr/bin/env python3
"""battery_client.py — 电池查询客户端"""

import rclpy
from rclpy.node import Node
from my_interfaces.srv import GetBattery


class BatteryClient(Node):
    def __init__(self):
        super().__init__('battery_client')
        # 创建客户端
        self.cli = self.create_client(GetBattery, 'get_battery')
        # 等待服务上线
        while not self.cli.wait_for_service(timeout_sec=1.0):
            self.get_logger().info('等待电池查询服务...')

    def send_request(self):
        """发送查询请求"""
        request = GetBattery.Request()  # 这个服务没有请求参数
        future = self.cli.call_async(request)
        return future


def main():
    rclpy.init()
    client = BatteryClient()
    future = client.send_request()
    rclpy.spin_until_future_complete(client, future)

    result = future.result()
    print(f'电池电量: {result.battery_percentage}%')
    print(f'电压: {result.voltage}V')
    print(f'充电中: {"是" if result.is_charging else "否"}')

    client.destroy_node()
    rclpy.shutdown()
```

### 21.12 TF2 坐标变换

> 🔰 **什么是 TF？** TF（Transform，坐标变换）解决的是这个问题：机器人身上有很多传感器（LiDAR、相机、IMU），每个传感器都有自己的坐标系。"LiDAR 看到障碍物在它前方 2 米"——能算出来这个障碍物相对于机器人底盘在哪里。TF 就是管理这些坐标系之间的关系。

```
Go2 的 TF 树示例：

         map
          │
          │  (地图坐标→里程计坐标)
          ▼
         odom
          │
          │  (里程计坐标→机器人坐标)
          ▼
       base_link
        │    │
     ┌──┘    └──┐
     ▼          ▼
  lidar_link  camera_link
  (LiDAR)     (相机)
```

#### 21.12.1 常用 TF 命令

```bash
# 查看当前的 TF 树
ros2 run tf2_tools view_frames
# 会生成一个 frames.pdf 文件，用图形显示 TF 树

# 查看两个坐标系之间的变换关系
ros2 run tf2_ros tf2_echo base_link lidar_link
# 输出: Translation: [0.0, 0.0, 0.45]
# 说明 LiDAR 在机器人底盘上方 0.45m 处
```

#### 21.12.2 发布静态 TF

```python
#!/usr/bin/env python3
"""
static_tf_publisher.py — 发布 Go2 上传感器的固定位置关系

比如 LiDAR 安装在 Go2 背部上方 0.45m 处，
这个位置是固定不变的，所以用"静态 TF"发布。
"""

import rclpy
from rclpy.node import Node
from tf2_ros import StaticTransformBroadcaster
from geometry_msgs.msg import TransformStamped


class StaticTFPublisher(Node):
    def __init__(self):
        super().__init__('static_tf_publisher')

        # 创建静态 TF 广播器
        self.broadcaster = StaticTransformBroadcaster(self)

        # 定义 base_link → lidar_link 的变换
        t = TransformStamped()
        t.header.stamp = self.get_clock().now().to_msg()
        t.header.frame_id = 'base_link'    # 父坐标系（机器人底盘）
        t.child_frame_id = 'lidar_link'    # 子坐标系（LiDAR）

        # 平移：LiDAR 在底盘上方 0.45m
        t.transform.translation.x = 0.0    # 前后偏移 (m)
        t.transform.translation.y = 0.0    # 左右偏移 (m)
        t.transform.translation.z = 0.45   # 上下偏移 (m)

        # 旋转：LiDAR 没有旋转（和底盘方向一致）
        t.transform.rotation.x = 0.0
        t.transform.rotation.y = 0.0
        t.transform.rotation.z = 0.0
        t.transform.rotation.w = 1.0       # w=1 表示没有旋转

        # 发布
        self.broadcaster.sendTransform(t)
        self.get_logger().info('已发布静态 TF: base_link → lidar_link')


def main():
    rclpy.init()
    node = StaticTFPublisher()
    rclpy.spin(node)
    rclpy.shutdown()
```

```python
#!/usr/bin/env python3
"""
go2_state_reader.py — 读取 Go2 运动状态

这个程序会订阅 Go2 的运动状态 Topic，打印出：
- 机器人位置 (x, y, z)
- 机器人速度 (vx, vy, vz)
- 机器人姿态 (yaw)
- 步态类型

运行前确保：
1. 已连接 Go2（网线连接，IP: 192.168.123.222）
2. 已 source unitree_ros2 环境
"""

import rclpy
from rclpy.node import Node

# 导入 Unitree Go2 消息类型
from unitree_go.msg import SportModeState


class Go2StateReader(Node):
    def __init__(self):
        super().__init__('go2_state_reader')

        # 订阅 Go2 运动状态 Topic
        self.subscription = self.create_subscription(
            SportModeState,            # 消息类型（Unitree 自定义的）
            'sportmodestate',          # Topic 名称
            self.state_callback,       # 回调函数
            10
        )
        self.get_logger().info('Go2 状态读取器启动！等待数据...')

    def state_callback(self, msg):
        """收到运动状态数据时的处理函数"""
        # 位置
        x = msg.position[0]     # 前后位置 (m)
        y = msg.position[1]     # 左右位置 (m)
        z = msg.position[2]     # 高度 (m)

        # 速度
        vx = msg.velocity[0]    # 前进速度 (m/s)
        vy = msg.velocity[1]    # 横移速度 (m/s)
        vyaw = msg.velocity[2]  # 偏航角速度 (rad/s)

        # 体高
        body_height = msg.body_height

        # 步态
        gait_type = msg.gait_type
        gait_names = {0: '空闲', 1: '快步', 2: '快跑', 3: '爬梯(正)', 4: '爬梯(反)'}
        gait_name = gait_names.get(gait_type, f'未知({gait_type})')

        # 打印
        self.get_logger().info(
            f'位置: ({x:.3f}, {y:.3f}, {z:.3f}) | '
            f'速度: ({vx:.2f}, {vy:.2f}, {vyaw:.2f}) | '
            f'体高: {body_height:.3f}m | '
            f'步态: {gait_name}'
        )


def main():
    rclpy.init()
    node = Go2StateReader()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

**第四步：编写 Go2 运动控制器**

创建 `go2_demo/go2_demo/go2_controller.py`：

```python
#!/usr/bin/env python3
"""
go2_controller.py — 用 ROS2 控制 Go2 运动

这个程序演示如何通过 ROS2 发送运动指令：
- 前进、后退
- 左右移动
- 左右转弯
- 特殊动作（打招呼、坐下等）

运行前确保：
1. 已连接 Go2
2. Go2 处于站立状态
"""

import rclpy
from rclpy.node import Node
import time

# 导入 Unitree API 消息类型
from unitree_api.msg import Request

# 导入 SportClient（用于构造运动请求）
# 注意：这里需要安装 unitree_sdk2_python
from unitree_sdk2py.go2.sport.sport_client import SportClient


class Go2Controller(Node):
    def __init__(self):
        super().__init__('go2_controller')

        # 创建 ROS2 发布者，发送 API 请求
        self.publisher = self.create_publisher(
            Request, '/api/sport/request', 10)

        self.sport_client = SportClient()

        self.get_logger().info('Go2 控制器启动！')

    def move(self, vx, vy, vyaw):
        """
        速度控制
        vx: 前进速度 (m/s)，正=前进，负=后退
        vy: 横移速度 (m/s)，正=左移，负=右移
        vyaw: 偏航角速度 (rad/s)，正=左转，负=右转
        """
        req = Request()
        self.sport_client.Move(req, vx, vy, vyaw)
        self.publisher.publish(req)
        self.get_logger().info(f'移动: vx={vx}, vy={vy}, vyaw={vyaw}')

    def stop(self):
        """停止运动"""
        req = Request()
        self.sport_client.StopMove(req)
        self.publisher.publish(req)
        self.get_logger().info('停止运动')

    def hello(self):
        """打招呼（挥手）"""
        req = Request()
        self.sport_client.Hello(req)
        self.publisher.publish(req)
        self.get_logger().info('执行: 打招呼')

    def sit(self):
        """坐下"""
        req = Request()
        self.sport_client.Sit(req)
        self.publisher.publish(req)
        self.get_logger().info('执行: 坐下')

    def stand_up(self):
        """站起来"""
        req = Request()
        self.sport_client.StandUp(req)
        self.publisher.publish(req)
        self.get_logger().info('执行: 站起来')


def main():
    rclpy.init()
    controller = Go2Controller()

    # 演示：让 Go2 做一系列动作
    controller.get_logger().info('=== 开始演示 ===')

    # 1. 前进 2 秒
    controller.move(0.3, 0.0, 0.0)   # 向前走 0.3 m/s
    time.sleep(2)

    # 2. 停下
    controller.stop()
    time.sleep(1)

    # 3. 原地左转 2 秒
    controller.move(0.0, 0.0, 0.5)   # 左转 0.5 rad/s
    time.sleep(2)

    # 4. 停下
    controller.stop()
    time.sleep(1)

    # 5. 打招呼
    controller.hello()
    time.sleep(3)

    # 6. 坐下
    controller.sit()
    time.sleep(3)

    # 7. 站起来
    controller.stand_up()

    controller.get_logger().info('=== 演示结束 ===')

    controller.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

#### 21.15.2 常见错误及解决

| 错误 | 原因 | 解决方法 |
|------|------|---------|
| `ros2: command not found` | 没有 source ROS2 环境 | `source /opt/ros/humble/setup.bash` |
| `Package 'xxx' not found` | 没有 source 工作空间 | `source ~/ros2_ws/install/setup.bash` |
| `Topic list 为空` | 没有连上 Go2 | 检查网线、IP、CycloneDDS 配置 |
| `colcon build 报错` | 缺少依赖 | `rosdep install --from-paths src -y` |
| `Publisher/Subscriber 收不到消息` | Topic 名称不一致 | `ros2 topic list` 检查 Topic 名 |
| `DDS 超时` | 网卡配置错误 | 检查 CYCLONEDDS_URI 中的网卡名 |

#### 21.15.3 ROS2 开发最佳实践

| 建议 | 说明 |
|------|------|
| **先仿真后实机** | 在 MuJoCo 仿真器中测试通过后再部署到真机 |
| **善用日志** | `self.get_logger().info/warn/error()` 打印关键信息 |
| **善用 rosbag** | 录制数据方便事后分析和调试 |
| **一个功能一个包** | 导航、控制、感知分开放在不同的 Package 中 |
| **善用 Launch** | 把所有启动流程写进 Launch 文件，一键启动 |
| **Topic 命名规范** | 使用 `/robot_name/功能/数据` 格式 |
| **参数化配置** | 把可变参数放在 yaml 文件或 Launch 参数中，不要写死在代码里 |
| **Ctrl+C 安全退出** | 确保程序能正确处理关闭信号，停止机器人运动 |
