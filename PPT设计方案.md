# 🎤 比赛路演 PPT 设计方案

> 适用于：中国计算机设计大赛 · 软件应用与开发赛道  
> 路演时间：建议 8-10 分钟（根据赛制调整）  
> 总页数：16-20 页

---

## 一、整体设计风格

### 视觉基调
- **主色**：深蓝 `#2563EB`（与产品一致，体现专业、科技感）
- **辅色**：金色 `#F59E0B`（用于高亮关键数据和创新点标注）
- **背景**：深色渐变（`#0F172A → #1E3A8A`）或纯白底 + 蓝色装饰
- **字体**：标题用「思源黑体 Bold」或「阿里巴巴普惠体 Heavy」，正文用「微软雅黑/思源黑体 Regular」
- **风格关键词**：科技感、AI 智能、清爽专业、数据驱动

### 版式原则
- 每页只传递 **1 个核心信息**
- 文字量控制在 **50 字以内/页**（配合演讲者口述）
- 大量使用**图标、示意图、流程图、截图**代替文字
- 关键数据用**超大号字体**展示（如 "1024维向量"、"1000+ 岗位"）

## 三、制作技巧与注意事项

### ✅ 演讲节奏建议

| 阶段 | 页码 | 时间 | 重点 |
|------|------|------|------|
| 开场 | 1-2 | 45s | 快速进入，不要啰嗦 |
| 痛点+方案 | 3-4 | 75s | 让评委了解做什么、为什么做 |
| 功能全景 | 5 | 45s | 快速概览 |
| **核心亮点** | **6-9** | **240s** | **评分重点！算法+AI 多花时间** |
| 技术细节 | 10-13 | 135s | 架构+安全+数据库 |
| 创新总结 | 14 | 45s | 强化记忆点 |
| 演示 | 15 | 120s | 最直观的展示 |
| 收尾 | 16-17 | 45s | 展望+致谢 |
| **合计** | | **~10min** | |

### ✅ 评委关注点（针对性准备）

| 评委可能提问 | 准备方向 |
|-------------|---------|
| "推荐算法和商业平台（BOSS 直聘）的区别？" | 强调面向高校场景、简历驱动而非投递驱动、行为实时学习 |
| "Qdrant 为什么不用 Elasticsearch？" | 原生向量支持、Cosine 距离优化、轻量级部署 |
| "AI 简历生成的质量如何保证？" | 分模块润色（不是一次全改）、模板保真、支持人工编辑 |
| "数据量大了怎么办？" | 连接池、缓存、向量索引、分页查询、可水平扩展 |
| "安全措施是否充分？" | JWT + bcrypt + Helmet + 速率限制 + 参数化查询 + CORS |
| "和现有就业平台的差异化？" | 高校定制化、教师咨询集成、考试日历、管理员零代码运营 |

### ✅ PPT 制作工具推荐

| 工具 | 适合场景 |
|------|---------|
| **Microsoft PowerPoint** | 功能全面，动画效果丰富 |
| **Keynote** | Mac 用户，设计感强 |
| **Figma** | 自定义设计最自由，导出PDF |
| **Gamma.app** | AI 辅助生成 PPT 初稿 |
| **iSlide 插件** | PPT 设计规范化 |

### ✅ 演示注意事项

1. **备用方案**：如现场演示，带一份本地部署版 + 录屏视频双保险
2. **网络**：赛场网络不稳定是常见问题，本地环境提前准备好
3. **字号**：正文不小于 20pt，标题不小于 32pt（投影仪清晰度有限）
4. **配色对比度**：投影环境偏亮，深色背景+白色文字 或 白底+深色文字效果最好
5. **动画克制**：使用简单的淡入/向上滑入，不要花哨转场
6. **页码**：每页右下角加页码，方便评委提问定位
