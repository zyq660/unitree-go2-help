## 📝 项目简介

**"琦"迹工作室**是一款面向高校学生的 AI 驱动就业辅助 Web 平台。平台深度融合**向量搜索引擎**、**大语言模型**与**协同过滤推荐算法**，为高校学生提供从求职信息获取、简历智能生成到教师一对一咨询预约的全链路就业服务。

### 解决的问题

| 痛点 | 解决方案 |
|------|---------|
| 就业信息分散，学生难以高效筛选匹配岗位 | AI 语义搜索 + 多维度向量推荐引擎 |
| 简历撰写能力不足，缺乏专业润色指导 | 通义千问大模型驱动的智能简历生成 |
| 教师咨询资源有限，预约流程不便 | 线上预约系统，自动化时段管理 |
| 就业数据统计滞后，缺乏可视化分析 | 实时数据看板，多维度统计图表 |
| 考试报名信息容易遗漏 | 150+ 重要考试日历与倒计时提醒 |

## ✨ 核心特色

### 1. 🧠 智能推荐引擎 — "千人千面"的岗位匹配
- **双通道推荐架构**：简历向量匹配 + 浏览行为协同过滤
- **1024 维文本向量**：基于通义千问 `text-embedding-v3` 模型
- **实时行为学习**：浏览 3 个岗位后即时更新推荐结果
- **推荐可解释性**：向用户展示匹配度百分比与匹配维度明细

### 2. 🤖 AI 简历生成 — 大模型赋能求职
- **通义千问 LLM 集成**：分模块专业化润色（项目经历 / 校园活动 / 职业技能 / 荣誉奖项）
- **保留原始排版**：基于 Word 模板插值，而非重新生成，确保格式美观
- **证件照嵌入**：支持上传一寸照片并嵌入到 Word 简历模板

### 3. 🔍 语义搜索 — 超越关键词的智能检索
- **向量化语义匹配**：输入自然语言描述即可检索语义相关岗位
- **同义词理解**："做网页的" 也能匹配 "前端开发工程师"
- **多维度聚合过滤**：语义搜索 + 行业/城市/学历条件联合筛选

### 4. 📊 数据驱动决策
- **就业数据看板**：多年度就业率趋势、薪资分布、行业流向
- **Chart.js 可视化**：柱状图 + 统计指标实时渲染
- **Excel 批量导入导出**：就业数据、岗位信息支持 Excel 模板化管理

## 🧠 AI 算法设计

### 1. 向量化推荐引擎

```
用户简历/行为 ──→ 通义千问 Embedding API ──→ 1024维向量
                                                │
                                                ▼
                                           Qdrant 向量库
                                                │
                                     ┌──────────┼──────────┐
                                     ▼          ▼          ▼
                                 向量近邻    维度匹配    协同过滤
                                 (cosine)   (加权求和)  (行为相似)
                                     │          │          │
                                     └──────────┼──────────┘
                                                ▼
                                          多维融合打分
                                                │
                                     ┌──────────┼──────────┐
                                     ▼          ▼          ▼
                                 行业去重    公司去重    分数阈值
                                (≤8/行业)  (≤3/公司)   (可配置)
                                     │          │          │
                                     └──────────┼──────────┘
                                                ▼
                                        最终推荐列表 (Top-K)
```

### 2. 推荐权重配置

| 维度 | 默认权重 | 说明 |
|------|---------|------|
| 向量语义相似度 | 0.50 | 简历/岗位向量 Cosine 相似度 |
| 多维特征匹配 | 0.50 | 以下各维度加权之和 |
| ├─ 行业匹配 | 0.30 | 目标行业与岗位行业是否一致 |
| ├─ 关键词匹配 | 0.25 | 简历关键词与岗位标签交集 |
| ├─ 城市匹配 | 0.20 | 目标城市与岗位所在城市 |
| ├─ 标签匹配 | 0.15 | 技能标签交集 |
| └─ 类型匹配 | 0.10 | 全职/实习/兼职 |

### 3. 行为时间衰减

| 时间窗口 | 衰减系数 | 含义 |
|---------|---------|------|
| 24 小时内 | 1.0 | 完全保留 |
| 3 天内 | 0.7 | 较高权重 |
| 7 天内 | 0.4 | 中等权重 |
| 7 天以上 | 0.2 | 低权重 |

### 4. AI 简历润色流程

```
原始简历数据 ──→ 分模块提取 ──→ 通义千问 LLM
                    │               │
          ┌─────────┼─────────┐     │
          ▼         ▼         ▼     ▼
      项目经历  校园活动  职业技能  荣誉奖项
          │         │         │     │
          ▼         ▼         ▼     ▼
      专业化描述  量化成果  标准化  分类整理
          │         │         │     │
          └─────────┼─────────┘     │
                    ▼               │
            Word 模板填充 ←─────────┘
                    │
                    ▼
            保留排版 + 嵌入证件照
                    │
                    ▼
              .docx 文件下载
```

## 🔐 安全防护体系

| 安全层 | 措施 | 说明 |
|--------|------|------|
| **传输层** | Helmet 安全头 | CSP / X-Frame-Options / HSTS |
| **认证层** | JWT Bearer Token | 7 天有效期，登录态管理 |
| **密码层** | bcrypt 10 轮加密 | 抗彩虹表攻击，密码强度校验（8 位 + 数字 + 大小写） |
| **访问控制** | 四级角色权限 | student → teacher → admin → super_admin |
| **速率限制** | express-rate-limit | 全局 500 次/15min，登录 20 次/15min |
| **文件安全** | 白名单扩展名 + 大小限制 | xlsx/pdf/doc/docx/图片，10MB 上限 |
| **注入防护** | 参数化查询 | 全部 SQL 使用 `?` 占位符，杜绝 SQL 注入 |
| **编码安全** | 文件名 UTF-8 修正 | 修复 multer 中文文件名乱码 |
| **CORS** | 白名单域名 | 仅允许指定前端域名跨域请求 |

```
users ──1:N──→ appointments
  │               ↑
  │               │
  │           teachers ──1:N──→ teacher_schedules
  │
  ├──1:N──→ employment_records
  │
  ├──1:N──→ knowledge_resources
```

## 🚀 快速开始

### 环境要求

- **Node.js** ≥ 18.0
- **MySQL** ≥ 8.0
- **Python** ≥ 3.8（简历生成功能需要）
- **Qdrant**（向量推荐功能需要，可选）

### 1. 克隆项目

```bash
git clone <repository-url>
cd new
```

### 2. 配置数据库

在 `server/config/index.js` 中配置 MySQL 连接信息：

```javascript
module.exports = {
    port: 3000,
    db: {
        host: 'localhost',
        user: 'root',
        password: 'your_password',
        database: 'employment_app'
    }
};
```

### 3. 初始化数据库

```bash
cd server
npm install
node init-db.js          # 创建表结构
node seed-jobs.js        # 生成 1000 条模拟招聘数据 + Qdrant 索引
```

### 4. 启动后端服务

```bash
cd server
npm start                # 默认监听 http://localhost:3000
```

### 5. 启动前端开发服务器

```bash
cd client
npm install
npm run dev              # 默认监听 http://localhost:5173
```

### 6. 访问平台

打开浏览器访问 `http://localhost:5173`

> **超级管理员默认账号**：详见数据库初始化脚本中的 seed 数据
